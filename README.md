# ABTest
ab实验分组、可创建年切、月切数据、随机分流、输出标签、准入标签配置、白名单等

项目简介
ABS（AB 实验 / 分流服务） 是一套面向业务的 A/B 实验配置与在线分流 后端系统，配套 联调与调试前端。它把实验按 数据域、分流字段、分组与权重 等进行建模，在运行时根据请求中的业务实体（如用户编号等）计算用户所在实验组并返回结果，并支持 版本、升版与分配记录 等规则，常与运营、风控等「对照组 / 实验组」场景集成。技术形态上为 Spring Boot + MyBatis + 缓存（本地 Caffeine / Redis）+ PolarDB，对外以 Dubbo Facade 为主，并提供 HTTP 测试/联调接口；另有 XXL-Job 等定时任务与 Nacos 等基础设施配合。

核心功能
方向	说明
实验生命周期	实验与分组的增删改查、分页与筛选；上线 / 暂停 / 结束 等状态流转；结束时间、长期对照等配置；升版与懒重分等相关开关（与产品规则一致）。
分流决策	单次分流、批量分流：按域、实验编码、业务实体等入参返回分组结果；运行中实验停止参与分流等策略（如 expStop 一类逻辑）。
配置维度	数据域 + 分流字段 枚举与绑定；分组权重、默认组、对照/实验组标识；白名单。
标签与准入	输出标签 / 准入标签（引用其他实验产出的标签定义）；上游实验变更前可查询 仍绑定该上游的下层实验，避免误操作。
观测与统计	分流侧计数等（如 Redis 维度的实验/分组计数，具体以后端实现为准）。
联调能力	abs-web：实验列表与编辑、上下线（含下层绑定确认）、单次/批量分流调试，对接 AbTestController、ShuntController 等 HTTP 路径。

模块结构（简）
abs-api：对外 Dubbo 接口与 VO（如 AbExperimentFacade、IShuntFacade、ShuntBatchFacade 等）。
abs-service：业务实现、Repository、HTTP Controller、缓存与任务等。
abs-mybatis-generator：MyBatis 代码生成。
abs-web（可选）：Vue 3 联调台，非运行时强依赖。
